ERP NAMING CONVENTIONS
======================

1. MODELOS
----------
- PascalCase
- Nombre singular
Ej: Company, Supplier, Customer, ThirdPartyTaxProfile

2. CAMPOS
---------
- snake_case
- Nombres claros y completos
Ej: tax_id, legal_name, accounting_start_date

3. RELACIONES
-------------
- ForeignKey: singular
- related_name: plural
Ej:
company = ForeignKey(Company)
related_name="activities"

4. ARCHIVOS
-----------
- models.py
- views.py
- forms.py
- admin.py
- urls.py

Si hay muchos modelos:
models/
    company.py
    activity.py

5. TEMPLATES
------------
/templates/app_name/model/action.html

Ej:
templates/sales/customers/list.html

6. URL NAMES
------------
app:model_action

Ej:
sales:customer_list
sales:customer_edit

7. VISTAS
---------
- Class-based views
- Nombre: ModelActionView
Ej:
CustomerListView
SupplierCreateView

8. FORMULARIOS
--------------
- Nombre: ModelForm
Ej:
CompanyForm
SupplierForm

9. CONSTANTES
-------------
- MAYÚSCULAS
Ej:
AFIP_CATEGORY_CHOICES

10. MIGRACIONES
---------------
- Nunca editar migraciones ya aplicadas
- Crear nuevas migraciones para cada cambio
